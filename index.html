1. เป้าหมาย ​

สร้าง Frontend สำหรับระบบแจ้งปัญหาและติดตามงานซ่อม จาก UI reference และ API contract ที่ได้รับ

ข้อสอบวัด:

  • การอ่าน UI
  • Component design
  • State และ form
  • API integration
  • Validation
  • Loading/error/empty state
  • Responsive layout

2. Technology ​

ต้องใช้:

  • React
  • Vite
  • JavaScript เท่านั้น
  • .js และ .jsx

ห้าม:

  • TypeScript
  • .ts, .tsx
  • Next.js
  • Hardcode ข้อมูลแทน API
  • เปลี่ยน API contract
  • ใช้ AI

3. งานที่ต้องทำ ​

FE-01 Ticket List ​

แสดง:

  • Title
  • Category
  • Location
  • Reporter
  • Priority
  • Status
  • Edit
  • Change status

ต้องมี:

  • Loading
  • Error + retry
  • Empty state
  • Responsive desktop/mobile

FE-02 Search and Filters ​

ต้องมี:

  • Search
  • Category filter
  • Priority filter
  • Status filter
  • Clear filters

FE-03 Create Ticket ​

Form:

  • title
  • description
  • reporterName
  • location
  • categoryId
  • priority

Validation:

  • required
  • title อย่างน้อย 3 ตัวอักษร
  • description อย่างน้อย 10 ตัวอักษร
  • reporterName อย่างน้อย 2 ตัวอักษร
  • location อย่างน้อย 2 ตัวอักษร

หลังสร้างสำเร็จ ต้องเห็นรายการใหม่โดยไม่ต้อง refresh เอง

FE-04 Edit Ticket ​

  • โหลดข้อมูลเดิม
  • แก้ไขและบันทึก
  • จัดการ not found
  • update UI หลังสำเร็จ

FE-05 Change Status ​

  • เปลี่ยน status ผ่าน API
  • มี loading/disabled ระหว่าง request
  • update UI หลังสำเร็จ

FE-06 Error Handling ​

รองรับ:

  • VALIDATION_ERROR
  • CATEGORY_NOT_FOUND
  • CATEGORY_INACTIVE
  • TICKET_NOT_FOUND
  • network error

FE-07 Summary ​

แสดง:

  • Open
  • In progress
  • Resolved
  • Active categories

4. UI Expectations ​

  • ไม่ต้อง pixel-perfect 100%
  • hierarchy อ่านง่าย
  • spacing สม่ำเสมอ
  • form label ชัดเจน
  • mobile 375px ใช้งานได้
  • status และ priority ต้องมีข้อความ ไม่ใช้สีอย่างเดียว
  • ไม่มี console error สำคัญ

Common Exam Policy ​

1. รูปแบบการสอบ ​

ข้อสอบแบ่งเป็น 2 ส่วนและสอบแยกกัน:

  • Frontend: React
  • Backend: Express + Prisma + PostgreSQL

ระยะเวลาสอบส่วนละ 5 ชั่วโมง

2. สิ่งที่อนุญาต ​

ผู้เข้าสอบสามารถ:

  • เปิดดูโปรเจกต์เก่าที่ตนเองเคยทำ
  • เปิดดู source code ของตนเอง
  • เปิดเอกสารทางการของ React, Express, Prisma, PostgreSQL และ JavaScript
  • ค้นหา syntax หรือ error message ผ่าน search engine
  • ใช้ editor, terminal, database client และ browser developer tools
  • ใช้จอเสริมได้ตามระเบียบของผู้คุมสอบ โดยทุกจอที่ใช้งานต้องสามารถตรวจสอบได้

3. สิ่งที่ห้าม ​

ห้ามใช้ AI ทุกประเภทระหว่างสอบ เช่น:

  • ChatGPT
  • Claude
  • Gemini
  • GitHub Copilot
  • Cursor AI
  • Windsurf
  • Codeium
  • Tabnine
  • AI extension หรือ AI autocomplete
  • AI search หรือ AI browser assistant
  • การให้ผู้อื่นช่วยเขียนหรือแก้โค้ด

ผู้เข้าสอบต้องปิดหรือ disable AI extension ก่อนเริ่มสอบ

4. การแชร์หน้าจอ ​

  • ต้องแชร์หน้าจอผ่าน Zoom ตลอดการสอบ
  • ต้องแชร์หน้าจอที่กำลังใช้งานจริง
  • หากใช้หลายจอ ต้องปฏิบัติตามวิธีที่ผู้คุมสอบกำหนด
  • ห้ามหยุดแชร์หน้าจอโดยไม่ได้รับอนุญาต
  • หาก connection หลุด ต้องแจ้งผู้คุมสอบทันที
  • ผู้คุมสอบอาจขอตรวจ browser tab, extension, process และ terminal history

5. การทุจริต ​

หากพบการใช้ AI รับความช่วยเหลือจากผู้อื่น หรือซ่อนอุปกรณ์/หน้าจอ:

  1. ปรับตกข้อสอบส่วนนั้น
  2. ยุติการสอบ
  3. ไม่สามารถสอบใหม่ได้ตามนโยบายของผู้จัดสอบ
  4. บันทึกหลักฐานไว้ในรายงาน

6. ภาษาโปรแกรม ​

ข้อสอบนี้ใช้ JavaScript เท่านั้น

  • Frontend: .js, .jsx
  • Backend: .js
  • ห้ามใช้ .ts, .tsx, TypeScript compiler หรือ tsconfig.json

7. การส่งงาน ​

ต้องส่ง:

  • Source code
  • README.md
  • .env.example
  • Migration และ seed สำหรับ Backend
  • Commit หรือไฟล์ส่งงานก่อนหมดเวลา

ห้ามส่ง:

  • node_modules
  • .env ที่มี secret จริง
  • API key หรือ password จริง

Domain Specification: Fixly ​

1. ภาพรวมระบบ ​

Fixly คือระบบแจ้งปัญหาและติดตามงานซ่อมภายในบริษัท

ตัวอย่างปัญหา:

  • เครื่องปรับอากาศไม่เย็น
  • หลอดไฟเสีย
  • คอมพิวเตอร์เปิดไม่ติด
  • อินเทอร์เน็ตใช้งานไม่ได้
  • โต๊ะหรือเก้าอี้ชำรุด

ผู้ใช้ต้องสามารถ:

  • ดูรายการแจ้งซ่อม
  • ค้นหาและกรองรายการ
  • สร้างรายการแจ้งซ่อม
  • แก้ไขข้อมูล
  • เปลี่ยนสถานะงาน
  • ดูข้อมูลสรุป

ไม่มี authentication ในข้อสอบนี้

2. Entity ​

Category ​

Field Type Required Description
id integer yes Primary key
name string yes ชื่อหมวด ไม่ซ้ำ
isActive boolean yes เปิดให้เลือกหรือไม่
createdAt datetime auto วันที่สร้าง
updatedAt datetime auto วันที่แก้ไข

Ticket ​

Field Type Required Description
id integer yes Primary key
title string yes หัวข้อปัญหา
description string yes รายละเอียด
reporterName string yes ชื่อผู้แจ้ง
location string yes จุดที่เกิดปัญหา
priority enum yes LOW, MEDIUM, HIGH, URGENT
status enum yes OPEN, IN_PROGRESS, RESOLVED, CANCELLED
categoryId integer yes Foreign key
createdAt datetime auto วันที่สร้าง
updatedAt datetime auto วันที่แก้ไข

3. Business Rules ​

BR-01 Category ​

  • Ticket ต้องอ้างถึง Category ที่มีอยู่จริง
  • สร้างหรือแก้ Ticket ให้ใช้ได้เฉพาะ Category ที่ isActive = true

BR-02 Validation ​

  • title: 3–100 ตัวอักษร
  • description: 10–1000 ตัวอักษร
  • reporterName: 2–80 ตัวอักษร
  • location: 2–100 ตัวอักษร
  • priority: ต้องเป็นค่าที่กำหนด
  • status: ต้องเป็นค่าที่กำหนด
  • categoryId: ต้องเป็น integer และมีอยู่จริง

BR-03 Default values ​

เมื่อสร้าง Ticket:

  • หากไม่ส่ง priority ให้ใช้ MEDIUM
  • หากไม่ส่ง status ให้ใช้ OPEN

BR-04 Delete ​

ไม่บังคับให้ลบ Ticket ในข้อสอบนี้
การยกเลิกให้เปลี่ยน status เป็น CANCELLED

API Contract — Fixly ​

Base URL:

text
http://localhost:4000/api

1. Response Shape ​

Single resource ​

json
{
        "data": {}
      }

Collection ​

json
{
        "data": [],
        "meta": {
          "page": 1,
          "limit": 10,
          "total": 24,

json

3. Tickets ​

GET /tickets ​

Query:

Name Type Description
search string title, description, reporterName
categoryId integer filter category
priority enum LOW, MEDIUM, HIGH, URGENT
status enum OPEN, IN_PROGRESS, RESOLVED, CANCELLED
sort string createdAt, priority
order string asc, desc
page integer default 1
limit integer default 10, max 100

Response item:

json

POST /tickets ​

json
{

Response:
