Fixly API Docs

เอกสารอ้างอิง API ของ be-exam API จริง สำหรับข้อสอบภาคปฏิบัติ Frontend (React) — ระบบ Fixly

เอกสารฉบับไฟล์เดียว (เปิดอ่านออฟไลน์ได้) · มิเรอร์จาก API Contract

ภาพรวม

Base URL https://fixly-backend-6pzm.onrender.com/api

Frontend เชื่อมกับ be-exam API จริง (deploy ไว้ที่ base URL ด้านบน — ใช้ร่วมกัน ไม่ต้องรัน backend เอง) ทุก endpoint (ยกเว้น /auth/*) ต้องแนบ Authorization: Bearer <token> — ถ้าไม่มี token หรือ token ไม่ถูกต้องจะได้ 401 UNAUTHORIZED ทุก path ด้านล่างเป็น path ที่ต่อจาก Base URL (เช่น GET /tickets = GET https://fixly-backend-6pzm.onrender.com/api/tickets)

ผู้ใช้มี role USER / ADMIN — บาง action (เช่น สร้าง ticket, ดูรายชื่อผู้ใช้) ทำได้เฉพาะ admin และข้อมูล ticket ถูกแยกตามผู้ใช้ฝั่ง server (ดู Tickets)

เอกสารนี้สรุปจาก API Contract, Domain Spec และ Sample Data — ใช้เป็นแหล่งอ้างอิงเดียวขณะต่อ be-exam API

ข้อตกลงร่วม

รูปแบบ Response

ทุก endpoint ตอบกลับด้วยรูปแบบที่สม่ำเสมอ 3 แบบ:

เดี่ยว (single resource)

{
  "data": {}
}

คอลเลกชัน (collection) — มี meta สำหรับ pagination

{
  "data": [],
  "meta": {
    "page": 1,
    "limit": 10,
    "total": 24,
    "totalPages": 3
  }
}

Error

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input.",
    "fields": {
      "title": "Title must contain at least 3 characters."
    }
  }
}

fields จะมีเฉพาะกรณี validation error (รายชื่อ field ที่ผิดพร้อมข้อความ)

Enum

Fieldค่าที่เป็นไปได้ค่า default ตอนสร้าง
priorityLOW, MEDIUM, HIGH, URGENTMEDIUM
statusOPEN, IN_PROGRESS, RESOLVED, CANCELLEDOPEN
roleUSER, ADMINUSER

Authentication

ขอ token ก่อนเรียก endpoint อื่น แล้วแนบ Authorization: Bearer <token> ทุก request (role ที่ได้กลับมาใช้กำหนดสิทธิ์ใน UI — สร้าง ticket = admin เท่านั้น)

POST/auth/loginเข้าสู่ระบบ รับ JWT

Request body

{
  "username": "demo",
  "password": "password123"
}

Response 200

{
  "data": {
    "token": "<jwt>",
    "user": { "id": 2, "username": "demo", "displayName": "Demo User", "role": "USER" }
  }
}
  • 401 INVALID_CREDENTIALS (username/password ไม่ถูกต้อง)

ใช้ user ที่ be-exam seed ไว้ login ได้เลย — admin/admin123 = ADMIN, demo/demo1234 = USER

Tickets

Visibility: ผลลัพธ์ถูกแยกตามผู้ใช้ที่ฝั่ง server — USER ได้เฉพาะ ticket ที่มอบหมายให้ตน, ADMIN ได้ทั้งหมด (frontend ไม่ต้องส่ง filter เพิ่ม)

GET/ticketsรายการ ticket (search / filter / sort / pagination)

Query parameters

NameTypeDescription
searchstringค้นหาจาก title, description, reporterName
categoryIdintegerกรองตาม category
priorityenumLOW, MEDIUM, HIGH, URGENT
statusenumOPEN, IN_PROGRESS, RESOLVED, CANCELLED
sortstringcreatedAt หรือ priority
orderstringasc หรือ desc
pageintegerdefault 1
limitintegerdefault 10, max 100

Response 200 — คอลเลกชัน + meta

{

      "priority": "HIGH",
      "status": "OPEN",
      "categoryId": 2,
      "category": {
        "id": 2,
        "name": "Facilities"
      },
      "assignedToId": 2,
      "assignedTo": {
        "id": 2,
        "username": "demo",
        "displayName": "Demo User"
      },
      "createdAt": "2026-06-20T04:00:00.000Z",
      "updatedAt": "2026-06-20T04:00:00.000Z"
    }
  ],
  "meta": { "page": 1, "limit": 10, "total": 24, "totalPages": 3 }

GET/tickets/:idรายละเอียด ticket

Path parameters

NameTypeDescription
idintegerid ของ ticket

Response

  • 200 เมื่อพบ — คืน { "data": { ...ticket } } (รวม category และ assignedTo)
  • 404 TICKET_NOT_FOUND (รวมกรณี USER ขอดู ticket ที่ไม่ใช่ของตน)
POST/ticketsสร้าง ticket (admin only)

Request body

{
  "title": "Air conditioner is not cold",
  "description": "The air conditioner in meeting room A is running but not cooling.",
  "reporterName": "Supreme",
  "location": "Meeting Room A",
  "priority": "HIGH",
  "categoryId": 2,
  "assignedToId": 2
}

ข้อกำหนด field (validation)

FieldRequiredกติกา
titleyes3–100 ตัวอักษร
descriptionyes10–1000 ตัวอักษร
reporterNameyes2–80 ตัวอักษร
locationyes2–100 ตัวอักษร
categoryIdyesinteger และต้องเป็น category ที่มีอยู่จริงและ isActive = true
assignedToIdyesinteger — id ของผู้ใช้ที่รับผิดชอบ (ต้องมีอยู่จริง); ดึงรายชื่อจาก GET /users
prioritynoenum priority (ไม่ส่ง = MEDIUM)
statusnoenum status (ไม่ส่ง = OPEN)

Response

  • 201 สร้างสำเร็จ — คืน { "data": { ...ticket } }
  • 400 VALIDATION_ERROR (รวมกรณีไม่ส่ง assignedToId)
  • 403 FORBIDDEN (ไม่ใช่ admin)
  • 404 CATEGORY_NOT_FOUND / USER_NOT_FOUND
  • 409 CATEGORY_INACTIVE
PATCH/tickets/:idแก้ไข ticket (partial update)

รองรับ partial update — ส่งมาเฉพาะ field ที่ต้องการแก้ (รูปแบบ field/กติกาเดียวกับ POST /tickets)
USER แก้ได้เฉพาะ ticket ของตน; เปลี่ยนผู้รับผิดชอบ (assignedToId) ได้เฉพาะ admin

Response

  • 200 สำเร็จ — คืน { "data": { ...ticket } }
  • 400 VALIDATION_ERROR
  • 403 FORBIDDEN
  • 404 TICKET_NOT_FOUND / USER_NOT_FOUND
  • 409 CATEGORY_INACTIVE (เมื่อแก้ไป category ที่ปิดใช้งาน)
PATCH/tickets/:id/statusเปลี่ยนสถานะอย่างเดียว

Request body

{
  "status": "IN_PROGRESS"
}

Response 200

{
  "data": {
    "id": 101,
    "status": "IN_PROGRESS"
  }
}
  • 400 VALIDATION_ERROR (status ไม่ถูกต้อง)
  • 404 TICKET_NOT_FOUND

Users admin only

GET/usersรายชื่อผู้ใช้ (สำหรับเลือกผู้รับผิดชอบ)

ใช้ทำ dropdown เลือกผู้รับผิดชอบตอนสร้าง/แก้ไข ticket — เรียกได้เฉพาะ admin

Response 200

{
  "data": [
    { "id": 1, "username": "admin", "displayName": "Admin User", "role": "ADMIN" },
    { "id": 2, "username": "demo", "displayName": "Demo User", "role": "USER" }
  ]
}
  • 403 FORBIDDEN (ไม่ใช่ admin)

Summary

GET/tickets/summaryตัวเลขสรุปสำหรับ dashboard

นับเฉพาะ ticket ที่ผู้เรียกเห็น (USER = ของตน, ADMIN = ทั้งหมด); activeCategories เป็น global เสมอ

Response 200

{
  "data": {
    "open": 5,
    "inProgress": 3,
    "resolved": 8,
    "activeCategories": 4
  }
}
